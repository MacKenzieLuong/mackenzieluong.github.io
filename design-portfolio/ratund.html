Project Overview

ratund is a chaotically fun rat-themed game created for Micro Jam 45, a 48-hour game jam centered around the themes "Miniature" and "Gambling Addiction." As the sole designer, I was responsible for all visual elements, including UI design, 3D texturing, and character design.

My Role: UI/UX Designer, 3D Texture Artist, Visual Designer

Tools: Procreate (for UI sketches and 2D art), Blender/3D Software (for texturing)

The Problem

Game jams present a unique design challenge: create compelling, cohesive visuals in an extremely compressed timeline while supporting gameplay that's intentionally chaotic. The game needed to:

  • Miniature Theme: Visually communicate scale and perspective to make players feel tiny
  • Intentional Chaos: Support difficult controls without frustrating players
  • Personality: Convey humor through over-the-top cheese aesthetics
  • Rapid Iteration: Allow for quick changes as gameplay mechanics evolved

Target Audience

Game Jam Players: Indie game enthusiasts who appreciate quirky, experimental games with unique mechanics and humor. They're forgiving of intentional jank when it serves the gameplay concept.

Constraints & Goals

‚è∞ Time Constraint

48-hour deadline required rapid prototyping and low-fidelity approaches to allow for iteration.

üéØ Learning Goal

First attempt at 3D texturing - needed to balance learning curve with deliverables.

üé® Visual Goal

Create memorable, humorous aesthetic that enhances the chaotic gameplay experience.

Research & Inspiration

After discussing with the team with our themes, miniature and gambling addiction, we went through a heavy brainstorming session and all agreed that we just wanted to make something silly and fun. We took inspiration from indie games that used intentional difficulty as a mechanic (e.g., Goat Simulator, Getting Over It) to add to the sillyness and a tiny rat stuck in a casino, where the player is addicted to the silly game loop.

Key Insight: Players embrace frustration when it's wrapped in humor and visual charm.

For the "miniature" theme, I researched exaggerated perspective and scale in games like Pikmin to understand how to make players feel small. We wanted to overexagerate the environment and ensure the player feels like the rat, small in an unknown casino, ready to mess around with whatever is interactable!

Design Process

01

Concept & Rapid Sketching (Hour 1-8)

Analyzed "Miniature" + "Gambling Addiction" themes and brainstormed concepts. Created 20+ low-fidelity sketches in Procreate exploring UI layouts, rat designs, and cheese aesthetics. Prioritized speed over polish to support rapid iteration and shared designs with the team to align vision.

02

UI Assets & Polish (Hour 8-14)

Designed over-the-top UI filled with rat and cheese motifs. Polished initial assets for clarity and playtested builds to ensure effectiveness. Created consistent visual language across all interface elements.

03

3D Texturing Attempt (Hour 22-26)

Learned 3D texturing basics while creating cheese textures for environment. Embraced imperfection - the slightly janky and painted textures actually enhanced the low poly, sketchy aesthetic we were going for.

04

Final Implementation & Testing (Hour 36-44)

Implemented and polished remaining UI elements. Consistent playtesting ensured proper rendering and visual quality. Made final adjustments to balance chaos with clarity across all game states.

Interface Design

UI Architecture

Design Challenge: Maintain UI readability in a deliberately chaotic game without interrupting the fun

Solution: UI elements remain completely unaffected and stagnant despite gameplay chaos. The game itself is intentionally not UI-heavy - chaos is expected and part of the enjoyment. Dialogue boxes and interactions stay clear, allowing players to focus on the mayhem without confusion.

Visual Effects - Drunk Mode

Design Challenge: Add content that isn't mechanic-heavy but amplifies the chaotic game style

Solution: When the rat gets drunk, the entire screen wobbles dramatically while the cheese meter (critical UI) stays stagnant. This optional gameplay element pokes fun at the chaos - "yes, it can get worse!" The rendering approach creates gameplay-affecting visual changes without compromising core UI readability.

Design Execution Under Constraints

Key Design Principles

  • Accessibility: Color + shape redundancy for critical UI elements; the rat/player was always visible and at a consistent location.
  • Rapid Iteration: Low-poly aesthetic wasn't compromise - it was strategic choice enabling fast iteration cycles under 48-hour deadline
  • Visual Feedback: Non-verbal state indicators (sound effects, moving objects/animations, line rendering shows what was being focused) reduce cognitive load during fast gameplay, even without clear visuals.

Key Design Decisions

üé® Low-Fi Aesthetic

Decision: Embrace sketchy, low-poly look with silly theme and visuals.
Rationale: Time constraints made polish impossible, so I leaned into a sketchy and silly style that fit the theme and chaotic nature of the game perfectly.

üßÄ Exaggerated Proportions

Decision: Make the environment comically oversized.
Rationale: Reinforces "miniature" theme by making the rat feel tiny in comparison to the world. We want to make sure that the miniature theme was not an oversight, but a deliberate design choice.

üòµ Limited UI Elements

Decision: Gameplay is chaotic and visual feedback is important.
Rationale: Reducing on-screen UI clutter allows players to better engage with the chaotic gameplay instead of quantitative completion and enhances overall immersion.

Results & Impact

Completed: All design deliverables within 48-hour deadline

Player Feedback: Players praised the humorous visual style, with many commenting that the "wobbly" aesthetic made the intentionally difficult controls more enjoyable
